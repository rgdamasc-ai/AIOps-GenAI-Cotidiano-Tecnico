# Engenharia de Prompt — Framework T-A-G

## Quando utilizar o Framework T-A-G (Task-Action-Goal)

O framework T-A-G (Task, Action, Goal) é ideal para contextos de alta complexidade, como análises de causa raiz, planos de execução e resolução de incidentes em produção que exijam metas mensuráveis. Não deve ser utilizado para tarefas triviais ou comandos diretos e simples que não demandam estruturação estratégica.

## Template

```markdown
# Task
[Espaço para texto]

# Action
[Espaço para texto]

# Goal
[Espaço para texto]
```
